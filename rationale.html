Welcome to Rationale page

Here we will list all the features and specifications implemented on the site as well as the function planned, but that could not be implemented. We will also justify our decisions for choosing the layout and design.


Implemented features:

About Homepage

  • We use a responsive design model, the page has two styles controlled by whether the screen size is bigger or smaller than 1024px
  • We design simple but elegant appearance to reflect professionalism.
  • we show our team-members' beautiful photos to make the page more lively and the client knows who we are.

About Protosite

  • protosite homepage page shows the main elements that the page has
  • lecture page we added a navigation aside bar with bookmarks of the document contents because lectures have a lot of information and is divided in subtopics. Also we added an optional feature to return the user to top of the page for commodity.
  • quiz page is implemented with jquery form ui plugin, which has beautiful appearance and powerful validation function.
  • resources page is designed to have special looks not just simple list the resources links, this page also have two styles for the responsive designed model
  • error page is designed to show when the user introduces a bad link.
  • The design of the site has the navigation bar at the top, so that the user can navigate the site whenever they want as it's a common trend these days. Originally we were using a Jquery but we changed it to a simple css trick we learned. This is because since part of the course introduces Jquery techniques, we wanted that final users of the site could demonstrate this Jquery technique in their assignments.

    Since some features like the aside navigation/link bar and the back to the top link may not be featured in the designated page we decided to use Javascript to detect their presence and in given case modify the width of the remaining elements.

    The style is full with css3 and html5 technology, like new html tags, border-radius, css properties abbreviation, etc.

    We conserved the pictures that were on the site because the questionnaire indicated so. Only we changed the colours of the banners so that they felt related to our homepage.

    We moved the contact link to the footer, and also added a site map on it. As it's a common feature nowadays and considering the site purpose and size (about 25 links distributed in 3 columns) we believed its a good idea.

    Finally we added a server functionality so that we did could manipulate the header and footer sections easily, and access to a error page of our own.
